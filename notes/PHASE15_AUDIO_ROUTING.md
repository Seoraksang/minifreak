# CM7→FX 오디오 라우팅 경로 분석 (정적 분석)

**Phase Audio Routing** | 2026-05-01 | 펌웨어 fw4_0_1_2229 바이너리 직접 스캔

---

## 개요

MiniFreak는 STM32H747 (CM4 + CM7 듀얼코어) + 별도 FX 코어의 3코어 구조이다.
CM7이 오디오 합성(DSP)을 담당하고, CM4가 I/O(UART, SAI, 디스플레이)를 담당한다.
FX 코어는 이펙트 프로세싱 전용이다.

> **교차참조**: [PHASE4_HARDWARE_MAPPING.md](PHASE4_HARDWARE_MAPPING.md) — 페리페럴 매핑

## FX 코어 DMA 상세

### DMA1 스트림별 레지스터 참조

| 레지스터 | 오프셋 | 참조 수 | STM32H7 매핑 |
|----------|--------|---------|-------------|
| `DMA1->LIFCR` | `+0x008` | — | Low Interrupt Flag Clear |
| `DMA1->HIFCR` | `+0x00C` | — | High Interrupt Flag Clear |
| `DMA1_Stream0->CR` | `+0x010` | **9** | Stream 0 Control |
| `DMA1_Stream1->PAR` | `+0x028` | **10** | Stream 1 Periph Addr |
| `DMA1_Stream2->M0AR` | `+0x040` | 1 | Stream 2 Mem Addr |
| `DMA1_Stream3->M0AR` | `+0x058` | 1 | Stream 3 Mem Addr |
| `DMA1_Stream4->M0AR` | `+0x070` | 1 | Stream 4 Mem Addr |
| `DMA1_Stream5->M0AR` | `+0x088` | 1 | Stream 5 Mem Addr |
| `DMA1_Stream6->M0AR` | `+0x0A0` | 1 | Stream 6 Mem Addr |

> **DMA1 Stream 0 (CR=9 refs)**: FX 코어 메인 DMA 전송 채널
> **DMA1 Stream 1 (PAR=10 refs)**: 페리페럴 주소 설정 (SAI2 또는 AXI SRAM)

### BDMA 참조

| 레지스터 | 참조 수 | 비고 |
|----------|---------|------|
| `BDMA->LIFCR` (+0x008) | 포함 | 7개 전체가 BDMA 베이스 참조 |

## FX 코어 SAI2 상세

| 레지스터 | 주소 | 비고 |
|----------|------|------|
| `SAI2->GCR` | `0x40015800` | 글로벌 컨트롤 |
| `SAI2->CHB->CR1` | `0x40015824` | Block B 제어 (마스터/슬레이브) |
| `SAI2->CHB->CR2` | `0x40015804` | Block B 설정 |

> FX 코어는 SAI2 Block B만 사용. CM4도 SAI2를 사용하므로 **물리적으로 동일한 SAI2 페리페럴 공유**.
> CM4=Block A(출력), FX=Block B(입력/출력)로 분할 가능.

## FX 코어 UART3

| 레지스터 | 주소 | 비고 |
|----------|------|------|
| `USART3->CR1` | `0x40004800` | 제어 레지스터 1 |

> CM4 UART3 (CR1=3 refs, CR3=1 ref)과 FX UART3 (CR1=1 ref) — **동일 UART3를 양쪽에서 액세스**
> CM4=TX(커맨드 전송), FX=RX(커맨드 수신)

## 추정 오디오 데이터 경로

```
┌─────────────────────────────────────────────────────────────┐
│                      CM7 (DSP 코어)                         │
│  오실레이터 → 필터 → VCA → NEON SIMD                         │
│       ↓                                                     │
│  AXI SRAM 0x240030E8 (버퍼 A, 8 refs)                       │
│  AXI SRAM 0x240061D4 (버퍼 B, 8 refs)                       │
│  HSEM 획득 → 쓰기 → HSEM 해제                               │
└───────────────────────────┬─────────────────────────────────┘
                            │ AXI SRAM 경유
┌───────────────────────────▼─────────────────────────────────┐
│                      CM4 (I/O 코어)                         │
│  HSEM 대기 → AXI SRAM 읽기                                  │
│       ├──→ SAI2 Block A → Audio Codec → MAIN OUT            │
│       │    (DMA2 Stream 0/4/7, 48kHz)                       │
│       │                                                     │
│       └──→ UART3 TX → FX 코어에 커맨드 전송                 │
│            (FX 파라미터 변경, 이펙트 on/off 등)               │
└───────────────────────────┬─────────────────────────────────┘
                            │ UART3 + AXI SRAM
┌───────────────────────────▼─────────────────────────────────┐
│                      FX 코어                                │
│  UART3 RX → 커맨드 수신 (CR1=1 ref)                         │
│  AXI SRAM 0x240003D0 (동기화 플래그, CM7과 공유)             │
│  DMA1 Stream 0 (CR=9 refs) → 오디오 데이터 전송              │
│  DMA1 Stream 1 (PAR=10 refs) → 페리페럴 주소                │
│  BDMA (7 refs) → 추가 전송                                  │
│  SAI2 Block B (3 refs) → FX 출력 또는 CM4으로 반환          │
│       ↓                                                     │
│  AXI SRAM 0x24010598 (CM4가 13 refs로 읽음)                 │
│  AXI SRAM 0x24012598 (CM4가 15 refs로 읽음)                 │
└─────────────────────────────────────────────────────────────┘
```

## 결론 (정적 분석 한계 내)

1. **CM7은 SAI/DAC 페리페럴을 직접 액세스하지 않는다** → 오디오를 AXI SRAM에 쓰고 CM4가 읽어서 출력
2. **CM4가 오디오 I/O의 중심 역할** — SAI2(6 refs) + DMA1(17) + DMA2(5) + BDMA(6)
3. **CM4↔FX 통신 = UART3 (커맨드) + AXI SRAM (데이터) + HSEM (동기화)**
4. **FX 코어는 DMA1(19 refs)이 가장 많은 참조** — 오디오 버퍼 DMA 전송이 주요 역할
5. **FX 코어 SAI2 Block B** 참조(3개) 존재 → FX가 자체적으로 오디오 출력하거나 CM4로 반환
6. **AXI SRAM 0x240003D0**이 CM7(2 refs)과 FX(4 refs) 모두에서 참조 → 코어간 동기화 핵심 플래그
