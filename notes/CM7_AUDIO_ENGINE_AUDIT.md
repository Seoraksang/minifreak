## 2. VTable 아키텍처 (핵심 발견)

### 2.1 메인 클러스터 (0x0806A7D8 - 0x0806C200)

**99개의 독립적인 vtable**이 연속으로 배치되어 있으며, 각 vtable은 **최소 1개 이상의 고유 메서드**를 가짐.

```
vtable 크기 분포:
  5 entries:  13 vtables (13 with unique methods)
  9 entries:   8 vtables
 11 entries:   4 vtables
 12 entries:   8 vtables
 16 entries:  23 vtables  ← 가장 큰 그룹
 18 entries:  17 vtables
 19 entries:   7 vtables
 20 entries:  19 vtables
```

### 2.2 기반 클래스 계층

vtable prefix 공유 분석:

| 공통 접두사 길이 | 서명 수 | 의미 |
|-----------------|---------|------|
| 19 functions | 2 | 동일 기반 클래스 (완전 공유) |
| 17 functions | 3 | 거의 동일 (1-2개 오버라이드 차이) |
| 13 functions | 4 | 같은 기반 + 파생 레벨 1 |
| 12 functions | 5 | 같은 기반 + 파생 레벨 2 |
| 3 functions | 10 | 같은 최상위 기반 (다양한 파생) |
| 2 functions | 4 | 최소 공통 인터페이스 |

→ **최소 3-4단계 상속 계층**이 존재 (Base → Mid → Derived → Specific)

### 2.3 기반 클래스 메서드 (가장 많이 참조된 함수)

| 함수 주소 | 참조 vtable 수 | 추정 역할 |
|----------|---------------|-----------|
| 0x0800B4F8 | 33 | **Init()** 또는 **Render()** - 핵심 가상 메서드 |
| 0x0800BAA4 | 27 | **Process()** 또는 **Render()** |
| 0x0800B4FC | 26 | Init/Render 변형 |
| 0x0800B4F4 | 23 | Init/Render 변형 |
| 0x0800B4F0 | 15 | 기반 클래스 유틸리티 |

| 0x0800C4E0 | 6 | 특정 엔진 그룹 공유 |

→ `0x0800B4F0-0x0800BAFC` 영역은 **오디오 엔진 기반 클래스**의 가상 메서드 테이블

## 5. 엔진 컴포넌트 구현 상태

### 5.1 오실레이터 엔진

**문자열 기반 확인: ❌ 불가** (stripped 바이너리 - 엔진 타입명 문자열 없음)

**vtable 기반 확인: ✅ 확정**

- 메인 클러스터에 **99개 독립 vtable** 존재
- 각 vtable이 고유 Init/Render 메서드 보유
- vtable 크기 16-20인 그룹이 59개 (오실레이터 + 필터 엔진 추정)
- vtable 크기 5-12인 그룹이 33개 (LFO, Envelope, 유틸리티 추정)

**VFP 연산 증거:**
- VSQRT×21: 주파수 계산 (오실레이터 pitch)
- VLDR×214: 방대한 계수 테이블 로드
- VABS×68: envelope, amplitude 처리
- 440Hz, PI, 2PI, LN2, LOG2E: 파형 생성 핵심 상수

### 5.2 필터 엔진

**증거:**
- 0x0800BEDC (4KB, VFP 밀도 최고 0.033): VLDR×50 + VABS×11 → 계수 로드 + 필터 상태
- 0x707 (1/√2) × 8회: Biquad/Bessel 필터 계수
- IIR 필터 명령어: VMLA.F32×4 (곱셈-누적)
- VDIV×13: 정규화 (필터 공식)

### 5.3 FX 엔진

**CM7에 FX 코드 없음 → FX 코프로세서 전담**

- CM7 → FX 참조: **967개** (0x08000000-0x0800FFFF)
- FX 0x0800_0000-0FFF: 30 refs (init/setup)
- FX 코어가 Reverb, Delay, Chorus 등 전체 FX 처리
- CM7은 오디오 생성 → FX 코어로 전송 → 결과 수신

### 5.4 Envelope / LFO

**증거:**
- 0.01 × 17회, 0.001 × 14회, 0.3 × 8회: 타임 컨스턴트 (Attack/Decay/Release)
- VABS×68: envelope 추종 (envelope follower)
- VSQRT×21: exponential curve 계산
- vtable 클러스터 내 소형 vtable (5-12 entries, 33개): LFO/Envelope 엔진 추정

### 5.5 모듈레이션 (Mod Matrix)

**증거:**
- VMUL.F32×6: depth × source value 곱셈
- 0x0802D37C (12KB): VFP multiply×151 → 모듈레이션 라우팅
- vtable 기반 다형성: 다양한 mod source/dest 처리

## 9. 99개 vtable → 엔진 타입 매핑 추정

vtable 크기와 VFP 사용 패턴을 기반으로 한 **추정** 매핑:

| vtable 크기 | 개수 | 추정 엔진 카테고리 | 비고 |
|------------|------|-------------------|------|
| 20 entries | 19 | **오실레이터 (OSC)** | 복잡한 Init+Render+Aux |
| 19 entries | 7 | **오실레이터 변형** | 기반+2 오버라이드 |
| 18 entries | 17 | **필터 / 오실레이터** | 중간 복잡도 |
| 16 entries | 23 | **필터 / FX Mod** | 기반+다수 오버라이드 |
| 12 entries | 8 | **Envelope / LFO** | 적은 메서드 |
| 11 entries | 4 | **LFO Shaper** | |
| 9 entries | 8 | **유틸리티** | |
| 5 entries | 13 | **단순 프로세서** | Quantizer, Shaper 등 |

**총합: 99개 vtable = 24 OSC + ~10 필터 + ~10 Envelope/LFO + ~20 유틸리티 + ~35 기타**

## 10. 남은 검증 과제

1. **vtable → 실제 엔진 타입 매핑**: Ghidra 디컴파일로 각 vtable의 Init/Render 함수 분석
2. **플라이츠(Plaits) 파생 여부**: 코드 패턴이 오픈소스 Plaits와 일치하는지 비교
3. **float 테이블 내용 분석**: 0x08034B20 영역이 실제 wavetable인지 확인
4. **dispatch 함수 식별**: 어떤 vtable이 선택되는지 결정하는 switch/case 함수 찾기
5. **voice 구조체 레이아웃**: 0x0803052C (73KB) 함수에서 voice 데이터 구조 추출
