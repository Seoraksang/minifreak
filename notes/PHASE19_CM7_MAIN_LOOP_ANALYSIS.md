## 2. CM7 main() 발견

### Reset → Init → Main Loop 체인

```
Reset Vector (0x08007F90)
  └→ FUN_08007f88 (56B) — 스택/힙 초기화, malloc 래퍼
       └→ FUN_08060674 (6B) — thunk: return *DAT_0806067c (함수 포인터)
            [4 callers: 다양한 HAL 유틸]
```

### CM7 main() = `FUN_08007e6c` (28B)

```c
void FUN_08007e6c(void) {  // main()
    FUN_08007bf4();   // init_1_hw: 하드웨어 초기화 (602B)
    FUN_08007b34();   // init_2: DMA/인터럽트 설정 (184B)
    FUN_08007af8();   // init_3: NVIC 우선도 설정 (56B)
    FUN_08007e88();   // init_4: NVIC 인에이블 (32B)
    FUN_0805d71c();   // init_5: 콜백 등록 + 초기 오디오 틱 (294B)
    while(true) FUN_0805d8cc();  // 메인 루프
}
```

## 3. 메인 루프 = `FUN_0805d8cc` (26B)

```c
void FUN_0805d8cc(void) {
    // 함수 포인터 2개를 간접 호출 (vtable 패턴)
    puVar1 = (uint *)FUN_08054284();    // 콜백 테이블 1
    (**(code **)*puVar1)();             // CM4→CM7 인터럽트 처리?
    puVar1 = (uint *)FUN_08054228();    // 콜백 테이블 2
    (**(code **)*puVar1)();             // 오디오 틱 콜백 (FUN_08034338)
}
```

> **핵심**: CM7은 폴링 기반 루프로 동작. TIM2 인터럽트가 아닌 **콜백 함수 포인터** 기반.
> `FUN_08054228()`이 반환하는 객체의 vtable[0]에 `FUN_08034338`이 등록됨.

## 4. Init 체인 상세

### init_1: `FUN_08007bf4` (602B) — 하드웨어 초기화
- DMB 동기화 (`FUN_08008230`)
- FPU 설정 (`FUN_08008274`)
- 캐시/TCM 설정 (`FUN_08008540`, `FUN_0800855c`)
- 클럭 설정 (`FUN_08008bdc` 850B — 복잡한 PLL 설정)
- GPIO 설정 (`FUN_08008610` 1458B)
- 메모리 매핑 (`FUN_08009358` 8KB — SRAM/TCM 레이아웃)
- NVIC 설정 (`FUN_08008f70`)

### init_2: `FUN_08007b34` (184B) — DMA/인터럽트
- `FUN_08008538` 호출: DMA 스트림 설정 (0x400바이트 크기)
- 레지스터 `+0xe0` 비트 조작: **NVIC ISER/ICER** (인터럽트 활성화)
  - 비트 0, 1, 2, 7 설정 → IRQ 0, 1, 2, 7 활성화

### init_3: `FUN_08007af8` (56B) — NVIC 우선도
- 레지스터 `+0xd8` 비트 0, 1 → **NVIC IP priority** 설정

### init_4: `FUN_08007e88` (32B) — NVIC 인에이블
- 레지스터 `+0xd4` 비트 0 → **SysTick 인에이블** 또는 NVIC ISER

### init_5: `FUN_0805d71c` (294B) — **콜백 등록 + 첫 오디오 틱**
- 8개 플래그 체크 후 함수 포인터 할당 (컨디셔널 콜백 등록)
- `FUN_08060e18(0xb, ...)`: IRQ 11 설정
- **`thunk_FUN_08034338()`**: **오디오 틱 함수 1회 직접 호출** (초기화)
- `FUN_0805d8e8`: 체인형 콜백 등록 (인자 다양: 1, 4, 5, 7, 8, 0xb, 0x10)
- 콜백 체인에 오디오 틱 + 기타 처리 등록

## 5. 오디오 틱 = `FUN_08034338` (5,046B) — **CM7 DSP 심장**

### 개요
- **0개 직접 호출자** (콜백으로만 호출됨)
- 9개 함수 호출
- 14개 do-while 루프
- 10개 vtable 간접 호출

### 호출 그래프
```
FUN_08034338 (audio_tick)
├── FUN_08034298()          — 공유 메모리 동기화
├── FUN_08054228()          — 콜백 테이블 가져오기
├── FUN_0800856c(2)         — 이벤트 플래그 설정 (event |= 2)
├── FUN_0800856c(4)         — 이벤트 플래그 설정 (event |= 4)
├── FUN_080543c4()          — ?
├── FUN_0805eaf0()          — float 계산 (atan2?)
├── FUN_08008118()          — 지연 대기 (SysTick 기반)
├── FUN_080081e4()          — NVIC IRQ pending
├── FUN_08008170()          — GPIO 비트 조작
├── FUN_08060b5c() ×11      — memset (버퍼 클리어)
└── [vtable calls ×10]     — 오실레이터/필터/FX 프로세싱
```

### 처리 파이프라인 (역추적)

```
1. 버퍼 초기화
   - 4개 오디오 버퍼 0x1AE바이트(430)씩 클리어
   - 0x1010101 패턴 (float 1.0f ×4 SIMD 초기화) ×23회
   - stride 0x360 → 4개 보이스 버퍼 (0x360 = 864바이트/보이스)

2. 파라미터 렌더링 (Smoothing)
   - 0x2E8, 0x5D0, 0x45C, 0x744 오프셋 → 4개 파라미터 블록
   - 패턴: value = target + -(target - prev) * rate
   - 값이 음수면 +1.0 → **fractal/wraparound** (위상/주파수)
   - 각 블록당 4개 서브파라미터 (0x88바이트 stride)

3. vtable 프로세싱 (오실레이터 렌더링)
   - stride 0xF82 → 보이스 오브젝트 크기 (3986바이트)
   - (*pcVar14)(puVar18) → 각 보이스의 프로세스 함수 호출
   - 루프: 보이스 배열 끝까지 (DAT_08034698)

4. 이벤트/트리거 처리
   - 0x81 바이트 → 트리거/게이트 플래그
   - 0x31(49)개 항목 복사 → **키/노트 이벤트 버퍼**

5. 오디오 라우팅/믹싱
   - vtable offset +0xC: 믹스/라우트 함수
   - vtable offset +0x14: 버퍼 할당
   - VectorFloatToUnsigned ×4: Q15→float 변환
   - FUN_0805eaf0: atan2/삼각함수 (FM 라디알?)

6. 출력 처리
   - GPIO 핀 토글 (FUN_08008170, FUN_080081e4)
   - NVIC 인터럽트 트리거 (CM4에 완료 알림)
   - SysTick 대기 (FUN_08008118): **샘플 간 타이밍**

7. 보조 파라미터 업데이트
   - 6개 × 0x130바이트 블록 → 6개 엔벨로프/LFO?
   - 3개 × 0x88바이트 → 3개 모듈레이션 소스?
```

### 메모리 레이아웃 (추정)

```
보이스 오브젝트 (0xF82 = 3986바이트):
+0x000: vtable 포인터
+0x014: 오실레이터 상태 (Q15)
+0x078: 서브모듈 vtable
+0x084: 웨이브테이블 데이터
+0x1C0: 파라미터 블록 1 (smoothing)
+0x2E4: smoothing rate
+0x2E8: smoothed value (위상?)
+0x2F8: target value
+0x304: rate
+0x458~0x478: 파라미터 블록 2
+0x5CC~0x5EC: 파라미터 블록 3
+0x740~0x760: 파라미터 블록 4
```

## 7. 오실레이터 트리플렛 (3종, 각 ~2.2KB)

| 함수 | 크기 | 호출자 | 특징 |
|------|------|--------|------|
| `FUN_0801afd0` | 2,244B | vtable | 기본 위상 누적 |
| `FUN_0801b8b0` | 2,260B | vtable | 동일 구조, 다른 offset |
| `FUN_0801da54` | 2,244B | vtable | 동일 구조, 다른 offset |

### 공통 구조
```
1. 활성 체크 (param_1+0x20, param_1+0x24)
2. 출력 버퍼 memset(0, 0x80) → 128샘플
3. 크로스페이드 상태 판별 (flag +0x4c, +0x74)
4. 위상 누적: position += increment × delta
5. 크로스페이드 범위: 0x3C0~0x640 (960~1600 샘플)
6. 최소 범위: 0x60 (96샘플)
7. 웨이브테이블 인덱스 계산 + 인터폴레이션
8. 출력 버퍼에 Q15 샘플 기록
```

## 12. C++ 재구현 로드맵 영향

### Phase 19 완료로 확정된 사항

1. **CM7은 폴링 기반 메인 루프** (TIM2 ISR 아님) → C++에서는 콜백/스레드로 대체
2. **오디오 틱 1회 = 1프레임 (128샘플 @48kHz ≈ 2.67ms)**
3. **보이스 구조체 크기: 3986바이트** → C++ struct로 재현 가능
4. **vtable 기반 다형성** → C++ 가상 함수로 자연 매핑
5. **4보이스 × 4파라미터블록 × smoothing** → 직접 구현
6. **Q15 고정소수점 ↔ float 변환** → SIMD 최적화 필요
7. **SysTick 기반 타이밍** → C++ `std::chrono` 또는 오디오 API로 대체

### 재현 난이도 재평가

| 컴포넌트 | 난이도 | 근거 |
|----------|--------|------|
| 메인 루프 구조 | ★☆☆ | 단순 while + 콜백 |
| 보이스 매니저 | ★★☆ | vtable 패턴, 3986B 구조체 |
| Smoothing 엔진 | ★☆☆ | 단순 1차 IIR 필터 |
| 오실레이터 코어 | ★★★ | 7.5KB, 웨이브테이블, 크로스페이드 |
| 라우팅/믹서 | ★★☆ | 8KB, 16채널 switch-case |
| 모듈레이션 매트릭스 | ★★★ | 7.6KB, 다중 소스/데스티네이션 |
| FX 엔진 | ★★★★ | 별도 펌웨어, 아직 미분석 |
