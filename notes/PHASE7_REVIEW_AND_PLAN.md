## 3. 프로세스 품질 평가

### 잘한 것 ✅
- **듀얼코어 역할 전환 발견** (Phase 4) — 초기 추정이 틀렸음을 과감히 수정
- **43-state FSM 완전 분석** (Phase 3) — 바이트 레벨 상태 머신 재구성
- **Phase 3↔5 교차 검증** — 펌웨어와 DLL이 동일 프로토콜 사용 확인
- **161 CC 매핑 완성** — 실시간 제어 가능한 파라미터 전부 식별
- **Collage 프로토콜 4서비스 도메인 완전 매핑** (Phase 5)
- **6경로 펌웨어 업데이트** — DFU/RK/Collage 전부 문서화

### 아쉬운 것 ⚠️
- Plaits 소스가 있음에도 실제 코드 diff를 안 함 (가장 가치 높은 분석)
- XRef 스캐너가 작동 안 해서 자동 네이밍 매핑 실패
- Phase 2에서 미완료 항목 3개를 Phase 4로 미루고 잊어버림
- 마스터플랜의 CM4/CM7 역할 오류를 완전히 수정 안 함

## 7. FX 코어 분석 배경 (왜 높은 우선순위인가)

MiniFreak의 사운드 엔진은 크게 두 부분으로 나뉩니다:
- **CM7 DSP** (512KB): 오실레이터 → 필터 → VCA (Phase 2-4에서 분석 완료)
- **FX 코어** (120KB): Chorus, Phaser, Flanger, Reverb, Delay, Distortion, Stereo (★ 미분석)

FX 코어는 **별도 DSP(DSP56362)에서 실행**되며 CM7과는 완전히 독립된 바이너리입니다. Phase 1에서 triage만 했고 심층 분석을 안 했는데:

1. **FX는 사운드의 절반을 차지** — 오실레이터/필터가 소스라면 FX가 최종 사운드를 결정
2. **FX 파라미터는 이미 매핑됨** (FX1-3, 33개) — CM4/CM7 쪽 파라미터는 알지만 FX 코어 내부 로직은 블랙박스
3. **커스텀 FX 가능성** — 딜레이 타임 확장, 리버브 알고리즘 수정, 새 FX 타입 추가 등
4. **120KB는 작은 바이너리** — 분석 시간 대비 가치가 높음
5. **DSP56362는 문서화된 아키텍처** — 24bit 고정소수점 DSP, 레지스터/명령어셋 공개됨
