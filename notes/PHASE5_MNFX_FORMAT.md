## .mnfx Format Specification

### Overview

- **Encoding:** boost::serialization text archive (single line, CRLF terminated)
- **Magic header:** `22 serialization::archive`
- **Archive version:** 10
- **Line termination:** CRLF (`\r\n`)
- **File structure:** Single line

### Header Structure

```
22 serialization::archive 10    -- boost::serialization magic + version
0 7                           -- tracking_level(0) class_version(7)
0 7                           -- tracking_level(0) class_version(7)
<len> <preset_name>           -- length-prefixed preset name string
<len> <bank_type>             -- length-prefixed bank type (e.g., "User")
66                            -- unknown integer (constant across all presets)
<len> <author_name>           -- length-prefixed author name string
<len> <category>              -- length-prefixed category string
<ints...>                     -- continuation fields (flags/state, ~24 values)
<len> Subtype <len> <value>   -- Subtype field (string value)
<len> Type <len> <value>      -- Type field (string value)
<ints...>                     -- pre-parameter integers (~13 values)
```

### Parameter Encoding

Parameters are encoded as sequential name-value pairs:

```
<name_length> <name> <value>      -- numeric parameter (int or float)
<name_length> <name> <val_len> <val>  -- string parameter (rare)
```

- **Integers:** Plain decimal (e.g., `0`, `-1`, `2368`)
- **Floats:** Plain decimal, IEEE 754 single precision (e.g., `0.71428573`)
- **Strings:** `<byte_length> <content>` (content may include spaces)
- **Parameter names:** Length-prefixed, contain underscores (e.g., `Arp_Mode`)
- **Order:** Alphabetical within sections (suggests `std::map` storage)

### Value Encoding Rules

- Single line format with CRLF line termination
- Parameter names are length-prefixed strings (boost::serialization std::string)
- Values follow immediately after parameter name
- Extra whitespace may appear between sections (boost::serialization padding)
- Parameters appear to be stored in a std::map (alphabetical order within sections)
